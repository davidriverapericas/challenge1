Planteamiento

Supongamos que necesitamos añadir a nuestro UI-KIt un nuevo componente de tipo live search input que usaremos en los formularios de nuestro producto. Para ello se requiere una solución web, compatible con cualquiera de los navegadores que actualmente están en el mercado y que tenga la siguiente funcionalidad:

  1. El usuario podrá escribir en el input y este llamará a un servicio de backend con esa información como parámetro. Los resultados se mostrarán filtrados.
  2. El tipo de filtrado debe ser de tipo contains y case insensitive El objetivo es que nos enseñes cómo gestionas el desarrollo de un elemento de interacción complejo por ti mismo, no puedes usar librerías que implementen todo el autocomplete (bootstrap typeahead, twitter typeahead, Downshift, ...) pero puedes usar librerías de apoyo para resolver el problema (React, jQuery, ...).

Requisitos

  • Repositorio remoto código fuente
  • Uso de cualquier api de pruebas como backend (puedes usar una API pública cualquiera, no necesitas crear tu propio endpoint).
  • Compatible con cualquiera de los navegadores que actualmente están en el mercado

Documentación de la Solución desarrollada

Pasos a seguir para incluir un elemento de tipo live search input en una pagina:

  1. Incluir los estilos del UIKit en el tag <head> de la página:
    <!-- UI Kit Styles -->
    <link href="/assets/css/uikit/uikit.css" rel="stylesheet">
  2. Incluir el Javascript del UIKit al final del tag <body>, para evitar bloquear la renderización de la página:
    <!-- UIKIT JS -->
    <script src="assets/js/uikit/uikit.min.js"></script>
  3. Incluir tantos elementos <input type="text" id="myuniqueid"> como se desee.
    Cada elemento debe tener su propio Id único, ejemplos:
    <h2>Encuentra profesionales de confianza:</h2>
    <input type="text" id="livesearch1" placeholder="Qué necesitas...">
    ...
    <h2>País de nacimiento:</h2>
    <input type="text" id="livesearch2" placeholder="Selecciona el país de nacimiento...">
  4. Inicializar los elementos live search input en tu código Javascript de la siguiente forma:
    var liveSearchComponent1 = new LiveSeach(elementId, apiurl, minTypedLength, userDefinedFormatFunction, userDefinedSelectedFunction);
     Descripción:
    • elementId: [string] Identificador único del elemento input en el DOM.
      Este parámetro es obligatorio
    • apiurl: [string] URI al Endpoint que proporcione datos en formato JSON. A esta URI se le concatenará la cadena introducida por el usuario en el input.
      Este parámetro es obligatorio
    • minTypedLength: [integer] Número de carácteres necesarios para hacer la llamada al Endpoint.
      Este parámetro es opcional. Valor por defecto: 2
    • userDefinedFormatFunction: [function] Función de usuario Javascript para dar formato a los datos. Esta función recibirá un parámetro con cada uno de los datos recibidos por el API que la función puede manipular. Esta función deberá devolver un string que se mostrará al usario.
      Este parámetro es opcional
    • userDefinedSelectedFunction: [function] Función de usuario Javascript que el componente live search input ejecutará una vez que el usuario seleccione uno de los resultados.
      Este parámetro es opcional

    Ejemplo #1:
    var liveSearchComponent1 = new LiveSeach(
        'livesearch1',
        '/fake-api/fake-data.php?format=plain&search=',
        1,
        null,
        function(res) {
            document.getElementById('livesearch1_result').innerHTML = res;
        }
    );
    Ejemplo #2:
    var liveSearchComponent2 = new LiveSeach(
        'livesearch2',
        '/fake-api/fake-data.php?search=',
        2,
        function(obj) {
            return (obj.name + ' <span class="ls_email">' + obj.email + ' de </span><span class="ls_company">' + obj.company + '</span>');
        },
        function(res) {
            document.getElementById('livesearch2_result').innerHTML = res;
        }
    );
